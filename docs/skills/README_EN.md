# Memory Palace Skills Docs

This directory describes the skills / MCP orchestration setup for Memory Palace.

If you only start `Dashboard / API / SSE` through `docker compose` or GHCR images, this is not always your first stop - continue only if you also want `Claude / Codex / Gemini / OpenCode / IDE hosts` wired in. Docker runs the service side; it does not automatically modify local skill / MCP / IDE host configuration.

If you want AI to guide installation step by step, install the standalone [`memory-palace-setup`](https://github.com/AGI-is-going-to-arrive/memory-palace-setup) repo, then say:

```text
Use $memory-palace-setup to install and configure Memory Palace step by step.
Prefer skills + MCP.
```

If you only want to manually connect a client to the Docker-exposed `/sse`, see:

- `docs/GHCR_QUICKSTART_EN.md`
- Section `6.2 SSE mode` and `6.3 client configuration examples` in `docs/GETTING_STARTED_EN.md`

## Canonical Bundle

The real canonical bundle is at:

```text
docs/skills/memory-palace/
├── SKILL.md
├── references/
├── variants/
└── agents/openai.yaml
```

Public docs explain how to use it. The canonical bundle defines what the skill actually is. `AGENTS.md` ships as a repo-local rule entry, allowing clients like Antigravity that support `AGENTS.md` to read repository constraints directly. Legacy setups can still fall back to `GEMINI.md`.

`Cursor / Windsurf / VSCode-host / Antigravity` no longer use hidden skill mirrors as the default integration. They consume the repository through `AGENTS.md + python scripts/render_ide_host_config.py --host ...`.

## Local Validation Reports

- `TRIGGER_SMOKE_REPORT.md` - Generated by `python scripts/evaluate_memory_palace_skill.py`
- `MCP_LIVE_E2E_REPORT.md` - Generated by `cd backend && python ../scripts/evaluate_memory_palace_mcp_e2e.py`

These are local review artifacts, not primary entry docs. The public repository excludes them via `.gitignore`. Before forwarding, check the content for local paths or client config traces.

For isolated output in parallel review or CI, set `MEMORY_PALACE_SKILL_REPORT_PATH` / `MEMORY_PALACE_MCP_E2E_REPORT_PATH`. Relative paths land under the system temp directory's `memory-palace-reports/`; absolute paths outside the repository give you full control.
